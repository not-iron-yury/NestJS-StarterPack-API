1. Роль AppConfigProvider

Разделение ответственности :

  .env
  |
  env.schema.ts → отвечает за валидацию
  |
  Env → тип для .env
  |
  configuration.factory.ts → отвечает за преобразование
  |
  AppConfig
  |
  AppConfigProvider → только предоставляет готовый AppConfig


AppConfigProvider не создает конфигурацию. У него нет права читать process.env, валидировать данные, вычислять значения.
Его задача предоставить готовый объект AppConfig через Dependency Injection NestJS.

--------------------------------------------------------------------------

2. Зачем нужен @nestjs/config и кастомный AppConfig?


@nestjs/config дает нам инфраструктуру.

Он умеет:
  - загружать .env;
  - запускать нашу фабрику;
  - хранить конфигурацию внутри Nest container;
  - внедрять конфигурацию через DI.


То есть:

@nestjs/config — это механизм доставки.

AppConfig - кастомная модель данных.
